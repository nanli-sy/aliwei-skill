# 逐条交付模板

## 核心原则

**禁止藏缺口**：每条任务都必须有明确的完成状态和详细说明，禁止含糊其辞。

## 状态定义

| 状态标识 | 含义 | 说明 |
|----------|------|------|
| ✅ 已完成 | 任务已全部完成 | 符合预期，无需额外操作 |
| ⚠️ 部分完成 | 任务完成了一部分 | 有未解决的问题或需要用户确认 |
| ❌ 未完成 | 任务完全未完成 | 无法实现或需要用户决策 |
| ⏸ 本期不做 | 本期暂不执行 | 非技术原因，如优先级调整、资源限制等 |

## 禁用词列表

交付时禁止使用以下含糊不清的表述：

| 禁用词 | 替换为 |
|--------|--------|
| "应该没问题" | "已验证通过，结果如下：..." |
| "差不多了" | "已完成，详细说明如下：..." |
| "应该可以" | "已验证，结果如下：..." |
| "可能行" | "已尝试，结果如下：..." |
| "做不了" | "已尝试 X 种方法，无法实现，原因如下：..." |
| "有点复杂" | "遇到技术难点，已尝试 X 种方法，详细分析如下：..." |
| "大概完成了" | "完成状态：✅ 已完成 / ⚠️ 部分完成，详细说明如下：..." |
| "基本实现了" | "完成状态：✅ 已完成 / ⚠️ 部分完成，详细说明如下：..." |
| "好像可以" | "已验证，结果如下：..." |
| "可能需要" | "建议：..." |

**核心原则：** 交付时必须给出明确的结论和验证结果，禁止使用模糊的表述。
