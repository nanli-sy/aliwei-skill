# 正面示例：首页性能优化

## 场景

用户要求："帮我优化一下首页性能，现在加载太慢了。"

---

### 第一阶段：意图识别与假设披露

**一句话总结意图：** 优化首页加载性能，提升用户体验。

**功能点拆解：**
1. 分析首页性能瓶颈
2. 实施前端性能优化
3. 验证优化效果
4. 提供优化报告

**🔴 关键假设披露：**

1. 我假设你说的"加载太慢"指的是首屏加载时间超过3秒，因为行业标准是2秒以内
2. 我假设需要优化的是前端性能，不包含后端优化，因为你没有提到后端
3. 我假设可以使用懒加载、代码分割、图片优化等常见前端优化手段
4. 我假设首页包含大量图片和JavaScript，这是常见的性能瓶颈

**🟡 约束清单：**

- 不能引入新的依赖包
- 必须保持现有功能不变
- 优化后需要通过回归测试

**🔵 歧义点确认：**

1. 是否有特定的性能指标需要达到？
2. 是否可以修改后端接口？
3. 是否需要考虑移动端优化？
